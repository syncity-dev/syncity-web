# Syncity Web — Claude Code Guide

## Project

Marketing and team website for Syncity, a company of independent software consultants.
Single-page layout: Header → Hero → Members → TechStack → ContactUs → Footer.

---

## Stack

| Layer | Technology |
|---|---|
| Framework | React 19 + TypeScript |
| Routing / SSR | TanStack Router + TanStack Start (NOT Next.js) |
| Styling | **Panda CSS** (NOT Tailwind) |
| Headless components | Ark UI |
| Images | `@unpic/react` (NOT `next/image`) |
| Icons | `lucide-react` + `react-icons` |
| Forms | react-hook-form + Zod + Formspree |
| Package manager | pnpm |
| Build | Vite + Nitro |

## Project Structure

```
src/

  components/
    core/            # Atoms — Button, Card, Heading, Text, Link, Section, TextInput, etc.

  utils/             # seo.ts, colorMode.ts
  styled-system/     # AUTO-GENERATED by Panda — never edit directly
```

## Component Architecture Rules

### Layer rules

| Layer | What belongs here | What does NOT belong |
|---|---|---|
| `core/` | Pure atoms with no business logic. Anything used in 2+ features. Has its own recipe. | Feature-specific components, data fetching |
| `shared/` | Layout primitives used on every page (Header, Footer, PageContainer, RootLayout) | Page-specific sections |
| `features/` | Page sections and their sub-components. Can compose from `core/` and `shared/`. | Reusable atoms |

### File pattern for core atoms

Core components with visual variants keep the component file in `core/` and the recipe in `src/recipes/`:

```
src/components/core/ComponentName/
  ComponentName.tsx         # React component
src/recipes/
  componentName.ts          # defineRecipe or defineSlotRecipe, registered in panda.config.ts
```

- **Single-element atoms** → `defineRecipe()` in `src/recipes/componentName.ts`, registered under `recipes` in `panda.config.ts`
- **Compound components** (multiple named slots, like Card) → `defineSlotRecipe()` in `src/recipes/componentName.ts`, registered under `slotRecipes` in `panda.config.ts`; use `createStyleContext` from `@/styled-system/jsx` in the component file

Run `pnpm prepare` after adding or changing any recipe in `src/recipes/`.

### Styling inside components

- Use `styled` from `@/styled-system/jsx` for polymorphic styled elements
- Use `css()` from `@/styled-system/css` for one-off class strings
- Use `defineRecipe()` in `src/recipes/` for single-element recipes with variants (register under `recipes` in `panda.config.ts`)
- Use `defineSlotRecipe()` + `createStyleContext` for compound components that need slot context (like Card) (register under `slotRecipes`)
- Never use `style={{}}` for anything covered by tokens

## Panda CSS / Token System

### The golden rule

**Never hardcode a value that has a token.** No raw hex colors, no raw px shadows, no raw font families. Always use the token reference.

### Token layers (in `src/theme/`)

```
tokens/          raw primitives: colors (black/white), fonts, z-index, animations, easings, durations
colors/          color palettes: slate, blue, red, green, amber, sky (all Radix 12-step + alpha)
semanticTokens/  semantic aliases: bg, fg, border, accent, shadows, radii, easings
keyframes.ts     CSS keyframe definitions
layer-styles.ts  surface layer styles
text-styles.ts   typography scale
breakpoints.ts   custom breakpoints (adds 3xl: 1800px)
conditions.ts    custom Panda conditions
global-css.ts    :root CSS vars, html/body base styles
motion/          interactiveTransition, textTransition helpers
```

### Semantic color tokens (use these in components)

```
bg.default / bg.subtle / bg.muted
fg.default / fg.muted / fg.subtle
border.default / border.strong
accent.default / accent.emphasis / accent.muted / accent.fg
danger.* / success.* / warning.* / info.*
```

### Color palette interaction sub-tokens

For buttons, badges, and interactive elements, use the palette sub-tokens via `colorPalette`:

```
colorPalette.solid.bg / .solid.bg.hover / .solid.fg
colorPalette.subtle.bg / .subtle.bg.hover / .subtle.fg
colorPalette.surface.bg / .surface.border / .surface.fg
colorPalette.outline.border / .outline.bg.hover / .outline.fg
colorPalette.plain.bg.hover / .plain.fg
```

Set `colorPalette="accent"` (or `danger`, `gray`, etc.) on the component root and use `colorPalette.*` in the recipe.

### Layer styles (surface patterns)

Use these instead of writing raw bg/border/shadow on surface elements:

```
surfaceSubtle    — tinted bg, no border/shadow (alternate section bgs)
surfaceRaised    — solid bg + border (cards, inputs)
surfaceElevated  — solid bg + shadow (floating cards)
surfaceOverlay   — solid bg + medium shadow (dropdowns, menus)
disabled         — grayscale + reduced opacity
```

### Text styles

`textStyle` prop: `xs` `sm` `md` `lg` `xl` `2xl` `3xl` `4xl` `5xl` `6xl` `7xl` `label`

### Font tokens

```
fontFamily: "body"     → Saira (use for paragraphs, buttons, labels, inputs)
fontFamily: "heading"  → Saira Condensed (use for all headings)
```

### Shadows

`shadow`: `xs` `sm` `md` `lg` `xl` `2xl` `inset` `top`

### Radii

`rounded`: `l1` (xs) · `l2` (sm) · `l3` (md) · `full`

### Z-index

`zIndex`: `footer` (5000) · `header` (6000) · `dropdown` (7000) · `overlay` (8000) · `modal` (9000)

### Animations

`animation`: `fadeIn` `fadeOut` `enter` `exit` `slideInFromTop` `slideInFromBottom` `slideOutToRight` `spin` `infinite-scroll`

### Responsive syntax

```tsx
// Panda responsive object syntax
fontSize={{ base: '3xl', md: '4xl' }}
display={{ base: 'none', md: 'flex' }}
```

Breakpoints: `sm` 640px · `md` 768px · `lg` 1024px · `xl` 1280px · `2xl` 1536px · `3xl` 1800px

## Color Mode

Light/dark mode is driven by a `data-color-mode` attribute on `<html>`. Panda CSS conditions `_light` / `_dark` read this attribute.

- `useColorMode()` hook in `src/hooks/useColorMode.ts` — returns `{ preference, resolved, setPreference }`
- `COLOR_MODE_INIT_SCRIPT` in `src/utils/colorMode.ts` — inline script in `<head>` prevents flash
- Preference stored in `localStorage` under key `color-mode` (`'light' | 'dark' | 'system'`)
- All semantic color tokens already handle both modes — components should never branch on color mode manually

## SEO

`seo()` util is in `src/utils/seo.ts`. Called in `__root.tsx` `head()` for global meta. Per-route head can override via TanStack Router's `head()` on each route.

Currently outputs: `title`, `description`, `keywords`, `og:type`, `og:title`, `og:description`, `twitter:*` (when image provided).

When adding `og:image`, provide an absolute URL.

## Accessibility Conventions

- Heading hierarchy: `h1` in Hero, `h2` for each section title, `h3` and below inside sections
- `aria-label` on icon-only interactive elements
- `aria-hidden="true"` on decorative duplicates (e.g., the duplicate `<ul>` in the TechStack marquee)
- `_motionReduce` Panda condition used for animations — always provide a `_motionReduce` fallback on animated elements
- Focus ring: `_focusVisible` with `outlineColor: "accent.default"` — do not suppress focus rings

---

## SSR Notes (TanStack Start)

- The app is SSR via Nitro. Avoid `window` / `document` access outside `useEffect` or after a `typeof window !== 'undefined'` guard.
- `COLOR_MODE_INIT_SCRIPT` runs as an inline script before hydration to avoid color mode flash.
- Images via `@unpic/react` handle responsive sizing and lazy loading — use `layout="constrained"` for fixed-size images, `layout="fullWidth"` for full-container images.
